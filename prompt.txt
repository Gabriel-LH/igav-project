🚀 Prompt para el Agente: Refactorización a Clean Architecture
Contexto:
Tengo un archivo "orquestador" que mezcla lógica de negocio, instanciación de repositorios de infraestructura (Zustand) y múltiples casos de uso en una sola función. Quiero refactorizar esto siguiendo Clean Architecture y DDD.
Instrucciones de Refactorización:
Eliminar la Capa de "Orquestador": En Clean Architecture no existe una carpeta "orchestrator". Mueve esta lógica a un Caso de Uso Maestro en src/application/use-cases/ProcessTransaction.usecase.ts.
Inyección de Dependencias:
No instancies repositorios con new dentro de la lógica.
Define Interfaces (Contratos) para cada repositorio en la capa de domain/repositories.
El caso de uso debe recibir estas interfaces en su constructor.
Separación de Lógica Financiera:
Extrae la función getFinancials y los cálculos de overpayment a un Domain Service o un Value Object llamado TransactionFinancials dentro de domain/logic. El dominio debe ser el dueño de cómo se calcula el dinero.
Patrón Strategy para Tipos de Transacción:
Sustituye los bloques if (dto.type === "venta") por una interfaz de estrategia o delega cada lógica a su propio Caso de Uso especializado (CreateSaleUseCase, etc.), pero sin que el maestro dependa de implementaciones concretas.
Manejo de Transacciones (Unit of Work):
Implementa una interfaz IUnitOfWork en el dominio. Asegura que todas las operaciones (createOperation, processPayment, createSale) se ejecuten dentro de un bloque atómico para que si una falla, ninguna se guarde.
Estructura de Carpetas Final:
src/domain/repositories/: Solo interfaces.
src/application/use-cases/: El nuevo ProcessTransactionUseCase y los sub-casos de uso.
src/infrastructure/stores-adapters/: Mantén aquí las implementaciones de Zustand, pero que implementen las interfaces del dominio.
src/application/dtos/: Mueve aquí los interfaces de DTOs.
Entregable:
Genera el código del nuevo Caso de Uso Maestro y explícame dónde colocar cada pieza para que el dominio sea 100% puro (sin importar nada de infrastructure o zustand).
